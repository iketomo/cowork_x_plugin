# X記事魅力紹介本文生成

X記事機能で公開した長文記事を、別途の通常ポスト（キャプション/ティザー文）で魅力的に紹介するための本文を生成する。  
Premium+ユーザーの自動カード投稿に加え、**クォート/引用ポストで紹介文を投稿**することでクリック率・読了率・アルゴリズム優位性を高める実践パターンに対応。

## 目的

- X記事のインプレッション・クリック・ブックマーク・コメントを最大化
- 記事本体（タイトル・本文）と完全に分離した「紹介ポスト」専用

## コアフレームワーク（AIDA-X）

1. **Attention（注意）**：冒頭1-2行でフック（3秒でスクロール停止）
2. **Interest（興味）**：記事のユニーク価値をティザー（箇条書き推奨）
3. **Desire（欲求）**：なぜ今読むべきか（体験・数字・限定感）
4. **Action（行動）**：明確なCTA
5. **X最適化**：絵文字・改行・ハッシュタグ2-3個・会話誘導（アルゴリズムが好む）

## 実践パターン（高エンゲージメント例の共通点）

- **冒頭1-2行が命**：質問・痛み・数字・意外性・ストーリーでフック。タイトル直コピペはNG
- **価値の約束**：読むと「何が変わるか」「誰向けか」を明確に。箇条書きや「完全版」「無料公開」で欲求を刺激
- **信頼性注入**：個人体験・努力エピロード・「かなり力を入れた」を軽く入れる
- **CTA必須**：「読んでみて」「感想教えて」「チェックどうぞ」でエンゲージメント促進
- **視覚最適化**：絵文字・改行・ハッシュタグ2-3個。文字数120-220文字目安（リンク込み）

## 推奨5パターン（テンプレート）

ジャンルに合わせて `[]` 内を記事内容に置き換える。

### 1. 問題解決型（仕事術・効率化系に最強）

```

実は[意外なやり方]で[具体成果]達成できました！

その全手順をX記事にまとめました👇

[記事リンク] #ChatGPT #仕事術
```


## 実行フロー

### Step 1: 記事情報の収集

ユーザーから以下を取得する。不足があれば確認する。

- 記事タイトル
- 本文要約（2-5行程度）
- ターゲット読者
- キーポイント（箇条書き2-5点）
- 記事URL（あれば）
- ジャンル・トーン（任意）

### Step 2: パターン選択

記事のジャンル・内容に合ったパターンを1-2個選択する。

| ジャンル例 | 推奨パターン |
|-----------|-------------|
| 仕事術・効率化 | 問題解決型、リスト型 |
| 実績・成果報告 | 成果・数字型、告知型 |
| 新発見・意外性 | 好奇心・ギャップ型 |
| 体験談・ストーリー | ストーリー型 |
| 体系的ノウハウ | リスト型、問題解決型 |

### Step 3: AIDA-Xでドラフト作成

1. フックを最優先で設計（冒頭1-2行で止まる文に）
2. 記事タイトルを一部引用してSEO/親和性UP（丸パクリはNG）
3. 5パターンのテンプレートに当てはめて生成

### Step 4: X最適化チェック

- 絵文字：適度に使用
- 改行：視覚的に読みやすい区切り
- 文字数：120-220文字目安（リンク込み）
- CTA：最後に明確な行動喚起
- ハッシュタグ：2-3個

### Step 5: バリエーション出力

A/Bテスト用に**2-3案**を出力する。パターンやフックを変えてバリエーションを持たせる。

## 追加ルール

- **記事タイトルを一部引用**してSEO/親和性UP（完全コピペはNG）
- **画像添付推奨**：記事ヘッダー画像の再利用を案内
- **投稿後フォロー**：自リプでQ&A誘導を案内
- **NG**：タイトル丸パクリ、長文すぎ（250文字超）、押し売り感

## 出力フォーマット

各案ごとに以下を提示する：

1. **パターン名**（問題解決型 / 成果・数字型 / など）
2. **完成文案**（そのままコピペ可能な形式）
3. **文字数**
4. **補足メモ**（工夫点・A/Bテストの観点など）
