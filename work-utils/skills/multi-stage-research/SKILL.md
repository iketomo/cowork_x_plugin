# 調査レポート作成スキル

テーマ「$ARGUMENTS」について、体系的な調査とレポート作成を行う。

## 実行フロー

```
Phase 1: 設計 → Phase 2: 並列調査 → Phase 3: 統合 → Phase 4: ドラフト → Phase 5: 批判チェック+修正 → Phase 6: 最終化
```

## ルール

| 項目 | 内容 |
|------|------|
| 調査エージェント | 並列起動が可能なら並列、不可なら直列で実行 |
| 出力上限 | 調査md: 3000文字以内 / 最終レポート: 制限なし / ソースURL 3つ以内 |
| 追加調査 | 最大2ラウンド |
| 一覧データ | Excel（openpyxl）で作成。できるだけ1ファイルにまとめる |

## 手順

各フェーズの詳細手順・テンプレート・文体ガイドはリファレンスを参照:

- [phases.md](references/phases.md) — Phase 1〜6 の実行手順
- [writing-guide.md](references/writing-guide.md) — 調査指示テンプレート・批判チェックテンプレート・最終レポート文体ガイド

## 並列実行の判定

以下の順で利用可能な並列手段を判定し、最初に使えるものを採用する:

1. **Agent Teams**（TeamCreate / SendMessage）— 環境変数 `CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1` が必要
2. **Agent tool の並列呼び出し** — 単一メッセージ内で複数 Agent を同時発行
3. **直列実行** — 上記いずれも不可の場合

## コンテキスト管理

- 全ファイルを一度に読み込まない。サマリベースで統合し、不明点のみ原文参照
- 同じ観点を複数エージェントに重複調査させない
- 統合ファイルに調査結果を全文転記しない（サマリのみ）
- 追加調査は最大2ラウンド。判断基準:「この観点は最終レポートの質に影響するか？」
