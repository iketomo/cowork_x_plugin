# Circle日次レポート生成スキル（アイザック）

Supabaseの`cowork`プロジェクトからCircleコミュニティ「アイザック」のデータを取得し、
サブエージェント`circle-daily-analyzer`に日次レポート作成を委譲するためのスキルです。

## このスキルの役割

- Circle「アイザック」の**日次レポート生成の入口**となる
- Supabase MCPを使って、日次レポートに必要なCircleデータを取得する
- 取得したデータをサブエージェント`circle-daily-analyzer`に渡し、分析〜レポート生成を任せる
- ユーザーには、レポートの概要とレポートファイルパスのみを簡潔に返す

## トリガー例

- 「Circle日次レポート」
- 「コミュニティレポート」
- 「アイザックのレポート」
- 「今日のCircle」
- 「コミュニティの状況」
- 「メンバーの動向」
- 「未対応コメント確認」
- 「Circleの分析」

## データ取得の方針（概要）

Supabase上の`circle_*`テーブル群から、日次レポートに必要な情報だけを取得する。
具体的なSQLやレポート構成の詳細は、**サブエージェント側の定義に任せる**。

- 基本サマリー
  - メンバー数・投稿数・コメント数・いいね数
  - 24h / 7dアクティブメンバー数
  - （daily_snapshotsがあれば）前日比・アクティブ率
- 未対応コメント候補（最重要）
  - 「いけとも」以外のコメントで、いけともの返信がない投稿
  - `circle_comments.raw_json`を活用して、コメント本文・投稿タイトル・URLなどを取得
- 新規投稿（過去24〜48時間）
- エンゲージメント急上昇投稿（delta_likes / delta_commentsが大きい投稿）
- メンバー動向
  - 新規入会者
  - 週間アクティブ投稿者
  - 長期未ログインメンバー（離脱予兆）

## サブエージェントへの委譲

- 取得した各種SQL結果（またはそれに相当する構造化データ）をまとめて、
  サブエージェント`circle-daily-analyzer`に渡す。
- `circle-daily-analyzer`側で、
  - 分析
  - 未対応コメントの特定
  - 盛り上げアクション提案
  - Markdownレポートファイルの保存
  をすべて完了させる。
- メインスキルは、**サブエージェントから返ってきたレポートパスと要約情報だけをユーザーに伝える**。

## 入出力の想定

- **入力（ユーザー）**: 上記トリガー例に類する自然文リクエスト
- **内部入力（サブエージェント）**:
  - サマリー数値
  - 未対応コメント候補一覧
  - 新規投稿一覧
  - エンゲージメント急上昇投稿一覧
  - メンバー動向（新規入会者・アクティブ投稿者・長期未ログイン など）
- **出力（ユーザー）**:
  - 当日のレポート概要（未対応コメントの有無・主な気づき など）
  - レポートMarkdownファイルのパス

## 注意事項（スキル側）

- Supabaseプロジェクトやレポート出力先パスなどの具体的な接続情報は
  各種`config.local.md`を参照すること。
- 新規投稿・コメントが0件の場合も「0件」と明示し、データ取得エラーと混同しない。
- メンバーのメールアドレスなどの個人情報は、ユーザーに返すサマリには含めない。
- SQLの詳細・レポートフォーマット・アクション提案の観点などは、
  なるべくサブエージェント`circle-daily-analyzer`側の定義に寄せること。
