# line-message スキル

LINE公式アカウントの「関心引き出しメッセージ」を作成する専門コピーライティングスキル。
行動経済学・心理学の知見と**実配信のCTR分析データ**に基づき、**1画面（150〜300文字）で「気になる／興味ある」ボタンをタップさせるメッセージ**を設計する。

---

## 【最重要】実証済みの勝ちパターン（自社CTR分析より）

過去の個人イベント配信のCTR分析から導出された、最も効果が高いパターン。**毎回必ず最初に検討すること**。

### 勝ちパターン1: 「読者の内声代弁」冒頭（CTR最大2倍）

冒頭1行目で**読者が心の中で思っていることを言い切る**と、CTRが平均6%台→8〜13%に跳ね上がる。
情報告知やベネフィット直球より、共感ファーストが勝つ。

| 内声代弁の型 | 冒頭例 | 実測CTR |
|------------|-------|---------|
| **あるある共感型** | 「YouTubeで見ても、「へー」で終わっていませんか？」 | **13.1%** 🏆 |
| **チャレンジ疑問型** | 「Agent Skills、もう作ってますか？」 | 9.2% |
| **自問自答代弁型** | 「気になってるけど、まだ触れてない…」 | 8.1% |
| **現状課題問いかけ型** | 「手作業のままにしていませんか？」 | 7.7% |

**設計テンプレート**:
```
【冒頭】読者の内声を1行で代弁
　「〇〇してみたけど、〇〇で終わっていませんか？」
　「〇〇、気になってるけど、まだ〇〇してない…」
　「〇〇、もう〇〇してますか？」
　「〇〇のままにしていませんか？」

【本文】それ、もったいない。
　〇〇すれば、〇〇ができます。

【ボタン】「気になる」
```

### 勝ちパターン2: ボタン文言は「気になる」を最優先（CTR +4〜6pt）

| ボタン文言 | 平均CTR | 心理ハードル |
|-----------|--------|-----------|
| **「気になる」** | **13.1%（最高）** | 最低（受動的でOK） |
| 「興味ある」 | 6.5〜9.2% | 中（能動的な意思表示が必要） |
| 「カルーセル各選択」 | 3.5%以下 | 高（選択疲れ） |

**ルール**:
- ステップ配信導入用の関心表明CTAは **デフォルトで「気になる」を選択** する
- 「興味ある」は意思表示色を強めたい場合のみ使う
- 既存の汎用CTA（「無料で参加する」等）は従来通りでOK

### 勝ちパターン3: 1配信1イベントに集中（複数告知はCTR分散）

カルーセルで5イベント並べた配信は、合計クリック数こそ多いが**1イベントあたりCTR 3.5%以下**まで低下する。
1配信で訴求するイベントは原則1つに絞る。

### 負けパターン（避けること）

| パターン | 実測CTR | 理由 |
|---------|--------|------|
| 情報告知型冒頭（「〇〇をやります！」「5つやります！」） | 3.5% | 価値提供前にイベント告知から始まると反応が薄い |
| カルーセル複数イベント並列 | 各3.5%以下 | 選択疲れでCTRが分散 |
| ベネフィット→課題の順 | 6.5% | 共感から始まる方が強い |

## 心理トリガー（6種）

メッセージには必ず**2〜3個を組み合わせて**使用すること。

| No | トリガー名 | 理論的根拠 | 概要 | LINE応用方法 |
|----|-----------|-----------|------|-------------|
| 1 | **損失回避** | カーネマン＆トベルスキー「プロスペクト理論」1979 | 利益より損失を2〜3倍重く感じる | 「逃すと損」を強調 |
| 2 | **好奇心ギャップ** | ザイガルニク効果関連 | 情報の空白を埋めたい欲求 | 答えを明かさず誘導 |
| 3 | **希少性** | チャルディーニ『影響力の武器』1984 | 限られたものに魅力を感じる | 数量・期間・対象を限定 |
| 4 | **社会的証明** | チャルディーニ『影響力の武器』1984 | 他者の行動を参考に決める | 参加者数・満足度を提示 |
| 5 | **認知的不協和** | フェスティンガー 1957 | 矛盾する情報を解消したくなる | 常識に反するフレーズで注意を奪う |
| 6 | **ザイガルニク効果** | ツァイガルニク 1931 | 未完了の情報が気になる | 意図的に情報を途中で切る |

### 効果的な組み合わせ例

- **内声代弁 x 認知的不協和 x 損失回避**: 関心表明CTRが最も高い（実測13.1%）
- **損失回避 x 希少性 x 社会的証明**: 締切系の訴求に最強
- **好奇心ギャップ x 認知的不協和 x ザイガルニク効果**: 情報系コンテンツの誘導に最適
- **社会的証明 x 希少性**: セミナー集客の王道

### トリガー別の文面例

| トリガー | イベント用 | 書籍用 |
|---------|----------|--------|
| 損失回避 | 「今申し込まないと損します！次回は3ヶ月後」 | 「この本を読まないと毎月2万円の無駄遣いを続けることに...」 |
| 好奇心ギャップ | 「なぜこの方法を使った人だけが月収2倍に？」 | 「年収1000万の人が毎朝やっている"たった1つの習慣"とは？」 |
| 希少性 | 「【あと2時間で締切】先着20名限定」 | 「【本日限定】通常3,800円→1,980円 在庫残り7冊」 |
| 社会的証明 | 「すでに3,500名が参加！94%が期待以上と回答」 | 「Amazonレビュー★4.6（1,200件超）」 |
| 認知的不協和 | 「『私には無理』と思っていた副業で主婦が月30万」 | 「食事制限なし、運動なしで3ヶ月で12kg痩せた」 |
| ザイガルニク効果 | 「このセミナーで話す内容はここでは書けません」 | 「この本の序文だけで読者の90%が涙...第1章を読み終える頃には...」 |

## コピーライティング・フォーミュラ（3種）

用途に応じて最適なフォーミュラを選択すること。

### AIDA型（注意→興味→欲求→行動）

**最適な用途**: オンラインセミナー告知、イベント集客

- A（Attention）: 視線を奪う1行 → 「無料セミナー、あと3枠です」
- I（Interest）: 自分ごと化 → 「1日30分の作業でCV率が2倍に」
- D（Desire）: 未来を描く → 「売上10倍達成した事例を公開」
- A（Action）: 行動を促す → 「今すぐ無料で詳細を見る」

### PAS型（問題→扇動→解決）

**最適な用途**: 課題解決型の訴求、書籍紹介、実用コンテンツ

- P（Problem）: 共感する課題提示 → 「LINE配信しても反応率が3%以下...」
- A（Agitation）: 危機感を煽る → 「競合は平均15%。このまま放置すると...」
- S（Solution）: 解決策を提示 → 「反応率を5倍にする設計術を公開」

### 4U型（有益・緊急・独自・超具体）

**最適な用途**: 緊急性の高い訴求、無料資料配布、SNS短文

- Useful（有益）: 「CV率150%アップのノウハウ」
- Urgent（緊急）: 「残り48時間限定」
- Unique（独自）: 「業界初の手法」
- Ultra-specific（超具体）: 「3ステップ・10分で完了」

## CTA設計ルール

### 効果実証済みの改善施策

| 施策 | 効果データ | OK例 | NG例 |
|------|----------|------|------|
| テキストリンク→ボタン形式 | CTR **+28%** | ボタン形式のCTA | テキストリンク |
| 名詞→動詞に変更 | 成約率 **+336%** | 「問い合わせする」 | 「問い合わせ」 |
| 「あなたの」→「わたしの」 | CTR **+90%** | 「わたしの特典を受け取る」 | 「あなたの特典を受け取る」 |
| 「注文」→「見る」に変更 | CTR **+340%** | 「今すぐ見る」 | 「今すぐ注文」 |
| マイクロコピー追加 | CVR **+33%** | 「※1分で完了・登録不要」 | ボタンのみ |
| ベネフィット明示 | - | 「売上10倍の事例を見る」 | 「資料ダウンロード」 |
| 文字数13〜15字以内 | - | 「無料で事例を見る」(9字) | 「今すぐクリックして詳細を確認してください」(21字) |
| 緊急性ワード追加 | - | 「今すぐ無料で見る」 | 「無料で見る」 |

### CTA6原則（必ず守ること）

1. **ベネフィット明示** - 押した先で得られるものを言語化
2. **動詞で行動促進** - 「〜する」「〜を見る」
3. **緊急性追加** - 「今すぐ」等のワード
4. **ハードル低減** - 「無料で試す」「見てみる」
5. **1人称活用** - 「わたしの〜を受け取る」
6. **文字数13〜15字** - 一瞬で理解できる長さ

### 用途別CTA文言ベスト例

- **イベント告知**: 「残り3枠 - 無料参加する」「席を確保する（1分で完了）」「限定資料つき - 今すぐ申込」
- **書籍紹介**: 「試し読みする（無料）」「1章だけ今すぐ読む」「読者限定特典を受け取る」
- **関心表明（ステップ配信導入）**: 「**気になる**」（最優先・実測CTR最高）、「ちょっと見てみる」、「興味ある」（意思表示を強めたい時）

> **重要**: 関心表明用CTAは「気になる」をデフォルト選択すること。「興味ある」より心理的ハードルが低く、CTRが1.4〜2倍高い（実測値）。

### マイクロコピー（ボタン直下の補助テキスト・15文字以内）

| パターン | 例 |
|---------|-----|
| 不安解消型 | 「クレジットカード不要」「1分で完了」「いつでも解除OK」 |
| ベネフィット強化型 | 「参加者の92%が満足」「今だけ限定資料つき」 |
| 緊急性訴求型 | 「残り3枠」「あと12時間」「先着50名限定」 |

## 「売り込み感」を消す5つの技法

LINEでは売り込み感が出ると即ブロックされる。以下を必ず適用すること。

| 技法 | NG | OK |
|------|-----|-----|
| 共感から入る | 「最高の商品です！」 | 「毎日忙しくて時間がない...そんなあなたへ」 |
| ベネフィット優先 | 「最新AI搭載」 | 「1日30分の作業を5分に短縮」 |
| 正直に伝える | 「誰でも100%成功」 | 「87%が効果実感（※個人差あり）」 |
| ストーリーで語る | 「業界No.1の実績」 | 「広告費ゼロだった私が年商1億になるまで」 |
| ポジティブ未来 | 「失敗しますよ」 | 「3ヶ月後、自動で集客できる仕組みが完成」 |

### 追加NG集（実際の運用から学んだパターン）

| NG表現 | 理由 | 代替案 |
|--------|------|--------|
| 「NPS指数43.5」など業界指標 | 読者に意味が伝わらない・冷たい印象 | 参加者の生の声・満足度%に置き換える |
| 「これまでのイベントは〇〇ではなかった」 | 過去参加者が「意味なかった？」と感じる | 「過去の実績を今回に全部ぶつける」と転換 |
| 「正直に言います。〜ではありませんでした」 | 違和感・信頼低下リスク | ポジティブな文脈で今回の特別感を伝える |
| 3日目の過度な煽り（「今すぐ！残り〇席！」） | プレッシャーで逆効果 | 講師の人格・温度感で動かす |
| 情報告知から始まる冒頭（「〇〇をやります！」「5つやります！」） | 価値提供前の告知はCTR3.5%まで低下 | 「内声代弁」型の冒頭に変える |
| カルーセル複数イベント並列告知 | 選択疲れで1イベントCTRが分散（3.5%以下） | 1配信1イベントに絞る |
| ベネフィット→課題の順（「プロレベルが作れます。けど面倒ですよね」） | 共感ファーストより弱い（CTR 6.5%） | 「課題（内声代弁）→ベネフィット」の順に組む |

## ステップ配信ファネル設計

「興味ある」ボタンタップ後のステップ配信シナリオも求められた場合は以下の構造で提案すること。

| ステップ | タイミング | 内容 | 目的 |
|---------|-----------|------|------|
| 0通目 | 即時 | お礼 + 無料特典配布 | 信頼構築 |
| 1通目 | 1日後 | 課題の深掘り・共感 | 問題意識の強化 |
| 2通目 | 3日後 | 成功事例・体験談 | 可能性の提示 |
| 3通目 | 5日後 | 限定オファー + 最終CTA | コンバージョン |

### 「興味あり」後のステップ配信 実証済みフレームワーク（イベント集客版）

実際に高い成果を出した3通シリーズのパターン。セミナー・ハンズオンイベントの集客に有効。

#### 直後（即時送信）：概念説明 + FOMO

- **目的**: 「何ができるのか」を5行以内でシンプルに伝え、申し込みへの最初の一歩を促す
- **構成**: ①一言で概念を説明 → ②具体的な活用例を箇条書き → ③「その差が広がっている」危機感 → ④CTA
- **文字数・行数**: 5〜10行、シンプルに
- **ポイント**: 専門用語・NPS等の業界指標は使わない。具体例の箇条書きが「自分ごと化」を促す

**例（イベント告知）:**
```
「Agent Skills」って一言で言うと、
「AIに自分の業務を丸ごと覚えさせる仕組み」です。
・XやInstagramで自分っぽい投稿文と画像を作る
・PowerPointテンプレで定形資料を作る
・メール配信ソフトからメールを作る
こういう仕事、毎回ゼロから指示している人と、
1回設定して全自動の人。
その差、2026年に取り返せなくなります。
▼ 詳細・申し込み
https://example.com
```

#### 1日後：共感 → 実績の声 → 今回の特別感

- **目的**: 「参加する意味があるのか」という迷いへの共感 → 実績で信頼構築 → 今回の特別感でクロージング
- **構成**: ①「わざわざ参加する意味あるかな？」と読者の疑問を代弁 → ②参加者の生の声（短いbullet 3〜4件） → ③「今回はさらに」という特別感 → ④CTA
- **NG**: NPS・満足度スコアなど専門指標の数字は使わない。生の声だけで十分
- **NG**: 「これまでのイベントは〇〇ではなかった」と過去を否定する表現 → 「過去の実績を今回に全部ぶつける」とポジティブに転換すること

**例（イベント告知）:**
```
「わざわざ2時間、イベントに参加する意味あるかな？」
正直、そう思いますよね。
でも参加者はこう言っています。
・AIを使う人から"作る人"になれた気がします
・断念していたツールが、2時間で普通に使えた
・5,000円でこんなに安いと思ったセミナーは久々
満足度95%超。YouTubeで見るのと、
自分の手を動かすのでは、全然違います。
過去イベントで磨いてきた内容を、今回に全部ぶつけます。
気合の入れ方が、これまでと違います。
▼ 申し込み
https://example.com
```

#### 3日後：講師からの個人的メッセージ（熱量型）

- **目的**: 数字や実績より「一緒にやろう」という温かい熱量で、迷っている人の背中を最後に押す
- **構成**: ①「〇日経ちましたね」と個人的な語りかけ → ②「一つだけ伝えさせてください」 → ③本質的な価値を正直に伝える → ④「一緒にやりましょう」でクロージング
- **トーン**: 煽りなし・プレッシャーなし。講師の人格・温度感で動かす
- **NG**: 「今すぐ！」「残り〇席！」などの過度な煽り表現

**例（イベント告知）:**
```
「興味あり」を押してくれてから、3日経ちましたね。
まだ迷っているなら、一つだけ伝えさせてください。
〇〇は、作った瞬間から使えます。
難しくないし、プログラミングも要りません。
ただ、「一人でやろう」と思うと
なかなか手が動かないのも、よく知っています。
だから2時間、一緒にやりましょう。
当日、全部の質問に答えます。
作り終わって帰ってもらいます。
▼ 申し込みはこちら
https://example.com
```

## 配信チェックリスト

メッセージ作成後、以下を自己チェックして問題があれば修正すること。

**メッセージ全体**
- [ ] 1画面（150〜300文字）に収まっているか
- [ ] AIDA / PAS / 4U のいずれかの流れに沿っているか
- [ ] 心理トリガーを2〜3個組み合わせているか
- [ ] 3秒でベネフィットが理解できるか
- [ ] 売り込み感がなく、共感 or 好奇心から入っているか
- [ ] 具体的な数字が1つ以上入っているか
- [ ] **冒頭1行目が「読者の内声代弁」になっているか**（情報告知型・ベネフィット直球型ではないか）
- [ ] **1配信1イベントに集中しているか**（カルーセル複数告知になっていないか）

**CTAボタン**
- [ ] 動詞が使われているか
- [ ] 13〜15文字以内か
- [ ] 緊急性ワードがあるか
- [ ] ベネフィットが明示されているか
- [ ] **関心表明用なら「気になる」を最優先で検討したか**（「興味ある」より実測CTRが1.4〜2倍）

**マイクロコピー**
- [ ] 不安解消要素があるか
- [ ] 15文字以内か

## 実行手順

### ステップ1: ユーザーへの確認

以下の情報をユーザーに確認する（不足している場合のみ質問する）:

1. **配信の目的**: セミナー集客 / 書籍紹介 / 無料資料配布 / 関心表明（ステップ配信導入） / その他
2. **ターゲット**: 誰に届けたいか（業種・役職・課題など）
3. **訴求する商品・サービス・イベントの詳細**: 何を伝えたいか
4. **使える数字・実績**: 参加者数、満足度、販売実績など
5. **制約事項**: 締切日時、定員、価格など

ユーザーが最初からこれらの情報を含めて依頼してきた場合は、確認を省略してそのまま作成に進む。

### ステップ2: メッセージ作成

上記の知識体系（心理トリガー、フォーミュラ、CTA設計ルール、売り込み感排除技法）に基づき、メインエージェントが直接作成する。

- サブエージェントは使用しない（コピーライティング品質はメインエージェントの文脈内で確保する）
- 3パターンを出力フォーマットに従って提案する
- 各パターンは異なるフォーミュラ・心理トリガーの組み合わせを使う

### ステップ3: 配信チェックリストで品質確認

上記の配信チェックリストで各パターンを自己チェックし、問題があれば修正する。

### ステップ4: ブラッシュアップ

ユーザーからのフィードバックに応じて修正・改善する。
必要に応じてステップ配信シナリオも提案する。
