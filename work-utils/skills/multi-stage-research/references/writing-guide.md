# テンプレート & 文体ガイド

## 調査指示テンプレート

エージェントへの調査指示に使用する:

```
## 調査指示
- 親テーマ: [テーマ名]
- 担当観点: [具体的な説明]
- 調査の深さ: [概要把握 / 詳細調査 / 徹底検証]
- 出力先: [絶対パス]
- 出力フォーマット:
  - 冒頭にエグゼクティブサマリ（5行以内）
  - 本文は見出し構造（3000文字以内）
  - 重要な数値・事実にソース明記（3つ以内）
  - 末尾に「追加調査が必要な点」
```

## 最終レポート文体ガイド

### ビジネス読み物スタイル

東洋経済オンラインやNewsPicksの特集記事のように、**ビジネスパーソンが通勤中に読みたくなる文章**を目指す。

**基本原則:**
- 箇条書きの羅列ではなく、**文章として流れる構成**にする
- 冒頭で読者の関心を掴む**問いかけ・フック**から始める
- 抽象論で終わらず、**具体例・数字・固有名詞**を必ず交える
- 段落ごとに「だから何？」「なぜ重要？」に答える意識を持つ
- 一文は60字程度を目安に。長文の連続を避け、リズムを作る

### 構成パターン

```
1. フック（具体的な人物×場面の逸話、または意外な事実で引き込む）
2. ナットグラフ（記事全体の10-15%以内に配置。何の話か・なぜ重要か・読む価値を宣言）
3. 本論（ダイヤモンド型で展開。各セクションは具体→抽象または抽象→具体の往復。
   セクション末尾にブリッジ文。問い→部分的解→新たな問いの連鎖）
4. 統合・収束（各論点の有機的なつながりを示し、全体像を再構成する）
5. 回帰と展望（冒頭の逸話に戻り、旅を完結させる。読者への示唆で締める）
```

### ナットグラフの義務化

記事全体の10〜15%以内（冒頭フックの直後）に「ナットグラフ（核心段落）」を必ず配置する。
ナットグラフは以下の3つの問いに答えること:
1. この記事は何についての話か
2. なぜ今これが重要なのか
3. この記事を読むと読者は何を得られるか

### ダイヤモンド型構成

逸話で始まる → ナットグラフで文脈化 → 具体と抽象の往復で展開 → 冒頭の逸話に回帰して収束。
最終セクションでは冒頭の逸話・問いに戻り、読者に「旅が完結した」感覚を与えること。

### テンション・カーブ

- 各セクション末尾には次セクションへのブリッジ文を置く
  - 例:「では、この原則をAI文章に適用しようとしたとき、何が壁になるのか」
- 情報開示は段階的に。冒頭で結論をすべて言い切らない
- 「問い→部分的な解→新たな問い」の連鎖で各セクションを構成する

### 見出し設計

見出しは結論・主張を含む具体的な文にすること。
- NG: 「認知科学の知見について」「AIの問題点」
- OK: 「読者の脳は『ギャップ』に引き寄せられる」「AIが書く文章はなぜ退屈なのか」

見出しだけをスキャンして記事の論旨が把握できる情報密度を持たせること。

### テクニック記事も読み物として書く

テクニックやノウハウ中心の記事でも箇条書き・表の羅列で終わらせない:

1. 各テクニックは「具体的な場面（誰が・何に困って・どう解決したか）」から導入する
2. テクニックの説明は地の文で流し、コード例・表は補助として使う（主役にしない）
3. テクニック間にブリッジ文を置き、文脈をつなぐ
4. 見出しは番号振りではなく主張型にする
5. 「使いどころ」「落とし穴」「実測効果」は地の文に自然に織り込む

### 文章テクニック

- セクション冒頭は**具体的な場面描写や問い**で始める
- 数字は文脈と比較を添える（「XX億円」→「前年比XX%増のXX億円、これはYY市場全体のZZ%に相当する」）
- 専門用語は初出時に一言で噛み砕く
- 適度に短い段落を挟み、読みやすさを確保する
- 結論を最後まで引っ張らず、各セクションで小さな結論を提示する

### 禁止パターン

- 冒頭の「この記事ではXXについて解説します。まず…、次に…」型の構成予告
- 「〜と考えられる」「〜と思われる」の2文連続使用（断定できる箇所は断定する）
- 同じ文型（主語+動詞+目的語）の3文連続
- 抽象的な説明のみで具体例が1つもないセクション
- 各セクションの冒頭が「〜について」「〜に関して」で始まる構成
- 「〜について述べる」「以下にまとめる」などの報告書口調
- 箇条書きの多用（極力地の文で展開する）
- 主語のない文の連続

### レポートテンプレート

```markdown
# [読者の関心を引くタイトル]

[フック: 問いかけ、意外な事実、象徴的エピソードから始める。2〜3文で読者を引き込む]

## [背景セクションの見出し]

[なぜ今このテーマが重要なのか。市場環境、技術動向、社会変化などの文脈を提供する]

## [本論セクション1の見出し]

[調査で分かった主要な論点を、具体例と数字を交えて展開する]

## [本論セクション2の見出し]

[別の角度からの分析。比較、事例、データで裏付ける]

## [示唆・インパクトの見出し]

[読者にとっての意味。「だから何？」に答える]

## [展望の見出し]

[今後の見通し、注目すべき変化、次のアクション]

---

**出典・参考情報**
（主要ソースを全て）
```

### 文字数

テーマの深さに応じて自由。制限なし。
