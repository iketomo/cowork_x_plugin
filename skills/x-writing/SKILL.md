# 文章の要件

## 🎯 成功する投稿の基本構成（必須）

### 1. 【衝撃的フック】読者の注意を一瞬で掴む（冒頭1-2行）
- 🚨🚀💡などの目立つ絵文字で開始（ただし💥は避ける）
- 具体的な数字の「差」や「規模」を冒頭に配置
- 「緊急警告」「ついに来た」「〇〇な方へ」などの強い言葉を使用
- 驚き・緊急性・新しさのいずれかを必ず含める
- 例：「🚨日本企業の生成AI、推進度は海外並みなのに成果は圧倒的に低い現実」

### 2. 【共感できる課題提示】読者の日常的な悩みから入る
- 「ビジネスパーソンなら一度は経験したことがあるはず」
- 「こんなツールがあったら業務が楽になるのに...と思いながらも」
- 読者が「これ、まさに自分のことだ」と感じる導入

### 3. 【データ/事例による裏付け】信頼性を高める
- 📊📈などの絵文字で視覚的に分かりやすく
- 具体的な固有名詞（PwC調査、Claude、GPT等）を使用
- 数字は必ず比較対象と共に示す（日本13% vs 米英50%超）
- 実体験に基づくエピソード、失敗談や試行錯誤の過程を交える

### 4. 【Before/Afterの明確な対比】変化を視覚的に
- 「従来：〜」「これから：〜」の形式で対比
- 読者が変化の大きさを直感的に理解できるように
- 例：「従来：アイデア→エンジニア依頼→要件定義→開発期間→やっと完成」
      「これから：アイデア→チャットで説明→即座に完成🎯」

### 5. 【原因分析・詳細説明】なぜそうなのかを解説
- 🔍🤔などの絵文字で思考を促す
- 表面的な事実だけでなく、深い洞察を提供
- 読者の理解と納得感を深める

### 6. 【具体的なアクションプラン】今すぐできることを提示
- ✅1️⃣2️⃣3️⃣などの番号で整理（3-4個が最適）
- 「今すぐ取り組むべき〇つのアクション」として明示
- 実践可能な行動や具体的ステップ・ツール活用法
- 読者が「明日から何をすべきか」を明確に
- 一般的な読者むけ。あまり専門的すぎる提案にはしない

### 7. 【前向きな締めくくり】行動喚起
- ポジティブな未来像を提示
- 「時代の幕開け」「可能性は無限大」などの前向きワード
- 「5年後に後悔しないために、今すぐアクションを起こしませんか？」など時間軸を入れた行動喚起

## ❌ 必ず避けるべきパターン

### 1. **煽りすぎ・誇張表現を避ける**
- 「ヤバすぎます」「衝撃」「緊急警告」の乱用はNG
- 💥などの過度な絵文字は使わない

### 2. **個人的感想への偏りを避ける**
- 「正直驚きました」「感銘を受けた」が中心はNG
- 客観的価値提供を重視

### 3. **専門的すぎる内容を避ける**
- 一般ビジネスパーソンに響かない専門用語はNG
- 実用性が見えにくい技術詳細は避ける

### 4. **ネガティブトーンの強調を避ける**
- 問題提起だけで解決策が弱いのはNG
- 不安を煽るだけの内容は避ける

### 5. **推測的・曖昧な内容を避ける**
- 「〜かもしれません」の多用はNG
- 断定的なトーン（「〜です」「〜が判明」）を使う

## 文量・段落構成
- 段落数：最低4段落、最大8段落程度
- 段落ごとの文字数：1段落あたり約100文字前後で改行
- 全体的に充実した内容量を確保

## 絵文字使用のルール
- 節の始まりに絵文字を配置（🚨、🚀、💡、✅など）
- 重要ポイントの強調（✨、🎯、💪など）
- 1段落につき1-2個程度に抑える
- 過度な使用は避ける

## 差別化ポイント
- 単なる紹介で終わらず、なぜその対策が有効なのか理由も添える
- 「成長」「スキルアップ」「クリエイティビティ向上」など、工夫することによる副次的な恩恵を提示
- 事実＋独自の切り口（ターゲットの課題に接続）＋具体的なアクションプランの3点セット
- 単なるニュース解説者ではなく、読者の課題を解決に導く「分析担当」「戦略アドバイザー」としてのスタンス

# X投稿用画像生成機能

## 概要
X投稿の文章に合わせて、Gemini Nano Banana Pro（gemini-3-pro-image-preview）を使って1:1のイラスト画像を自動生成する機能。

## 使い方
投稿文章を作成した後、以下のスクリプトで画像を生成する：

```bash
# 直接テキストを指定
python scripts/generate_image.py "投稿テキスト"

# ファイルから読み込み
python scripts/generate_image.py --file post.txt

# 出力先を指定
python scripts/generate_image.py "投稿テキスト" --output my_image.png
```

## 環境変数
- `GEMINI_API_KEY`: Google Gemini APIキー（必須）

## 画像スタイル（固定）
スクリプト内にデザインスタイルが固定プロンプトとして埋め込まれており、毎回一貫したデザインで画像が生成される。

### スタイル概要
- 日本のビジネス書・解説本でよく使われる「ゆるいイラスト」スタイル
- 線：黒の細いペン線、手描き風
- 塗り：水彩風の淡い色（クリーム、薄い青、薄いオレンジ、薄い緑）
- 人物：2〜3頭身のデフォルメキャラ、丸い顔、シンプルな表情
- 背景：白（#FFFFFF）
- アクセント：薄い青、薄い緑、薄いピンク
- 強調：赤（×マーク）、緑（✓マーク）

### サイズ
- アスペクト比：1:1（正方形）
- 解像度：2K

## X投稿作成時の画像生成フロー
1. 投稿文章を作成（上記の文章要件に従う）
2. `scripts/generate_image.py` に投稿文章を渡して画像を生成
3. 投稿文章 + 画像をセットでXに投稿

## 依存パッケージ
```bash
pip install google-genai
```
